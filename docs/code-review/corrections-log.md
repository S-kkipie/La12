## Open discrepancies to decide

- **RESOLVED (fix #7):** `frontend/lib/format.ts` `formatDate` was migrated from `Intl.DateTimeFormat`
  to date-fns (`format(..., "d 'de' MMMM 'de' yyyy", { locale: es })`) ΓÇö output-equivalent to the old
  `es-Es` long form, and the `opts` param was unused by every caller. The one remaining `Intl` date
  helper is `formatLimaDateTime` (webinar cards), which is intentional: pinning output to
  `America/Lima` regardless of the viewer's tz needs `Intl`'s `timeZone` option (date-fns core can't
  without `date-fns-tz`); it is centralized in `format.ts`, not inline in a component. `formatSoles` /
  `formatNumber` stay `Intl` ΓÇö they format currency/numbers, not dates, so the date rule doesn't apply.
